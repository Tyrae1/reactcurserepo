1. Базовий роутінг

Зробити структуру роутів:

/ — Home
Короткий текст + лінки на інші сторінки:
“Список курсів”
“Профіль користувача”
/courses — список курсів
Є масив курсів у коді (id, title, level, category).
Вивести список курсів.
Кожен курс має лінк на деталі: /courses/:id.
/courses/:id — сторінка одного курсу
Показати детальну інформацію про курс.
/profile — умовна сторінка профілю
Просто текст + кнопка “Перейти до курсів”.
Обов’язково використати:

<BrowserRouter>, <Routes>, <Route>
<Link> або <NavLink> для навігації
2. Хуки React Router (обов’язкові)

На різних сторінках використати такі хуки:

useParams
На сторінці /courses/:id:
Витягнути id з URL.
Знайти курс у масиві по id та показати його дані.
useNavigate
На сторінці курсу:
Кнопка “Назад до списку” → navigate(-1) або navigate('/courses').
На сторінці /profile:
Кнопка “До курсів” → програмна навігація navigate('/courses').
useLocation
В Layout / Header:
Показати поточний шлях (наприклад, Current route: /courses).
Опціонально: підсвітити активний пункт меню, орієнтуючись на location.pathname, якщо не хочуть використовувати NavLink.
useSearchParams
На сторінці /courses:
Додати інпут search (пошук по назві).
Додати select level (наприклад: all, beginner, advanced).
При зміні фільтрів оновлювати search-параметри: ?search=react&level=beginner
При завантаженні сторінки брати значення фільтрів з searchParams і одразу застосовувати їх до списку.
3. Мінімальні вимоги до ДЗ (можете так і дати студентам списком)

Студент повинен:

Налаштувати роутер з маршрутами: /, /courses, /courses/:id, /profile.
Використати:
useParams на сторінці деталей курсу.
useNavigate для кнопок “Назад” / “Перейти до курсів”.
useLocation для відображення поточного шляху в шапці.
useSearchParams для фільтра/пошуку на сторінці /courses.
Зв’язати UI ↔ URL:
Фільтри на /courses мають читати і оновлювати searchParams.
Детальна сторінка курсу має працювати по id з URL.
Програмна навігація через useNavigate.
